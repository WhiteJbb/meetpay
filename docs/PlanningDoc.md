# 기획서: 모임 더치페이 정산 웹

## 1. 서비스 개요

모임(친구, 동호회, 여행, 회식 등)에서 발생한 지출을 기록하고, 참여자별
분담 방식을 자유롭게 지정해 최소 이체 횟수로 정산 결과를 자동 계산해
공유하는 웹 서비스. 회원가입 없이 링크만으로 접근 가능한 익명형
서비스로, 소규모 모임(2~10명 내외, 지출 수십 건)에 최적화한다.

배경과 목표의 상세 맥락은 [ProjectContext.md](./ProjectContext.md) 참고.

## 2. 10문10답 결정 사항 요약

| # | 질문 | 결정 |
|---|------|------|
| 1 | 정산 대상 범위 | 매 지출마다 참여자를 자유롭게 선택(부분 참여 기본 지원) |
| 2 | 로그인/인증 | 회원가입 없이 모임 링크만으로 접근(익명) |
| 3 | 참여자 식별 | 실명 사용, 동명이인은 이메일로 구분 |
| 4 | 실제 송금 연동 | 계산 결과만 제공, 송금은 사용자가 직접(계좌 복사 등) |
| 5 | 모바일/PC 우선순위 | 처음부터 동등한 반응형 지원 |
| 6 | 정산 결과 공유 | 누구나 볼 수 있는 공유 링크 |
| 7 | 통화 단위 | 다국통화 지원(환율 변환 포함, 해외여행 모임 고려) |
| 8 | 영수증 첨부 | 첨부 가능하지만 선택 사항 |
| 9 | 모임 규모 | 소규모(2~10명, 지출 수십 건) 위주 |
| 10 | 이력 보관/삭제 | 정산 완료 후 일정 기간이 지나면 자동 삭제 |

## 3. 핵심 기능 목록

1. 모임 생성 및 참여자 등록 (실명 필수, 이메일·계좌는 선택)
2. 모임 공유 링크 발급 (로그인 없이 접근)
3. 지출 항목 등록
   - 지출자, 금액, 통화, 항목명
   - 참여자별 분담 지정(전원 또는 일부 선택)
   - 영수증 이미지 첨부(선택)
4. 다국통화 처리
   - 지출 등록 시 통화 선택
   - 환율 변환을 통한 기준 통화 환산 정산
5. 정산 계산
   - 참여자 간 최소 이체 횟수로 "누가 누구에게 얼마" 계산
6. 정산 결과 공유
   - 공유 링크를 통한 결과 조회(참여자 외 열람 가능)
7. 이력 보관/삭제
   - 정산 완료 후 일정 기간 경과 시 모임 데이터 자동 삭제
   - 삭제 예정 안내(예: 만료일 표시)

## 4. 기능별 상세 요구사항

### 4.1 모임 생성 및 참여자 등록
- 모임 생성 시 모임명, 참여자 목록을 입력한다. 이름은 필수(실명),
  이메일과 계좌 정보는 선택 입력이다.
- 동명이인은 이메일로 구분한다. 이메일은 같은 이름의 참여자가 2명
  이상일 때만 필수로 전환된다. (2026-07-07 페르소나 검토 반영:
  "1분 내 모임 생성" 목표를 위해 상시 필수에서 완화,
  DecisionLog §7 참고)
- 계좌 정보(은행/계좌번호)는 정산 결과에서 "받을 사람" 계좌
  표시/복사용으로 선택 입력한다. (2026-07-07 페르소나 검토 반영)
- 참여자는 모임 생성 후에도 추가/수정/삭제할 수 있다(단, 이미 지출에
  포함된 참여자 삭제 시 처리 정책은 상세 설계 단계에서 결정).

### 4.2 지출 항목 등록
- 지출자(누가 냈는지), 금액, 통화, 지출 항목명, 지출일을 입력한다.
- 참여자 분담은 체크박스 형태로 개별 지정 가능해야 한다(기본값은
  전체 선택이되, 자유롭게 해제할 수 있어야 함).
- 영수증 이미지는 선택적으로 첨부 가능하다(업로드/스토리지 필요).

### 4.3 다국통화 처리
- 지출 등록 시 통화를 선택할 수 있어야 한다.
- 정산 계산 시 모임의 기준 통화로 환산하며, 환율 정보 출처와 갱신
  주기는 상세 설계 단계에서 결정한다.

### 4.4 정산 계산
- 모든 지출 항목을 바탕으로 참여자별 순 잔액(낸 돈 - 부담해야 할 돈)을
  계산한다.
- 순 잔액을 바탕으로 실제 이체가 필요한 참여자 쌍을 최소화하는
  알고리즘(예: 그리디 방식)으로 "A가 B에게 얼마" 형태의 이체 목록을
  생성한다.

### 4.5 정산 결과 공유
- 모임/정산 결과 조회용 공유 링크를 제공한다.
- 로그인 없이 링크만으로 참여자 외 인원도 조회 가능하다(익명 서비스
  특성상 링크 자체가 접근 권한이 되므로, 링크 유출 시 열람 가능함을
  사용자에게 안내해야 한다).

### 4.6 이력 보관/삭제 정책
- 정산 완료(또는 모임 마지막 활동) 시점 기준으로 일정 기간이 지나면
  모임 데이터를 자동 삭제한다. 구체적인 보관 기간은 상세 설계
  단계에서 결정한다(예: 90일).
- 삭제 예정 안내를 결과 화면에 표시한다.

## 5. 비기능 요구사항

- **보안**: 실명, 이메일, 계좌 정보 등 개인정보를 다루므로 공유
  링크는 추측 불가능한 형태(예: UUID)로 발급하고, 개인정보 보관
  기간을 명시한다. 계좌 정보는 선택 입력이며 모임 데이터 만료 시
  함께 삭제된다.
- **성능**: 소규모 모임(참여자 10명 이하, 지출 수십 건) 기준으로
  정산 계산은 즉시(수 초 이내) 완료되어야 한다.
- **반응형**: 모바일/PC 환경에서 동등한 사용성을 제공해야 한다.
- **확장성**: 실제 송금 연동, 로그인 기반 계정 시스템 등은 이후 확장
  범위로 남겨두되, 초기 데이터 모델이 이를 막지 않도록 설계한다.

## 6. MVP 범위 vs 이후 확장 범위

### MVP(1차 범위)
- 모임 생성/참여자 등록(실명+이메일)
- 지출 항목 등록(부분 참여 지정, 다국통화)
- 정산 계산(최소 이체 횟수)
- 공유 링크를 통한 결과 조회
- 영수증 첨부(선택)
- 기간 경과 후 자동 삭제

### 이후 확장 범위(MVP 이후)
- 실제 송금 API 연동(토스/카카오페이 등)
- 로그인/계정 기반 관리(모임 이력 개인별 조회 등)
- 정산 완료 후 지출 내역 수정/감사 로그
- 알림(카카오톡/문자 등) 자동 발송
