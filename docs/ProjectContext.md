# 프로젝트 배경 및 목표: 모임 더치페이 정산 웹

## 배경

모임(친구, 동호회, 여행, 회사 회식 등)에서 여러 명이 비용을 나눠 낼 때,
누가 얼마를 냈고 누구에게 얼마를 줘야 하는지 계산하고 공유하는 과정이
번거롭다. 흔히 메신저 대화방에서 수기로 계산하거나 엑셀/메모장을
돌려쓰는데, 다음과 같은 문제가 반복된다.

- 지출 항목이 여러 건일 때 1/N 또는 부분 참여 계산이 헷갈림
- 누가 누구에게 송금해야 하는지(정산 경로) 최소화가 안 돼 이체가 여러 번 발생
- 정산 내역이 대화방에 묻혀서 나중에 확인하기 어려움
- 참여자 중 일부만 특정 항목에 포함되는 경우 처리가 어려움

이런 반복적인 불편을 해결하기 위해 모임 단위로 지출을 기록하고
자동으로 정산 결과(누가 누구에게 얼마)를 계산해주는 웹 서비스를
만들고자 한다.

## 목표

1. 모임(그룹)을 만들고 참여자를 등록할 수 있다.
2. 지출 항목(누가 얼마를 냈는지, 누가 얼마씩 부담하는지)을 기록할 수 있다.
3. 기록된 지출을 바탕으로 최소 이체 횟수로 정산 결과를 자동 계산한다.
4. 정산 결과를 모임 참여자들과 쉽게 공유할 수 있다.
5. (범위 미확정) 로그인/인증, 실제 송금 연동, 모바일 대응 등은 이후
   기획 단계에서 구체화한다.

## 현재 상태

- 아직 구현 전 단계이며, 서비스 요구사항을 구체화하기 위한 기획
  프로세스를 진행 중이다.
- 진행 순서: (1) 배경/목표 정리 → (2) 기획자 서브에이전트의 10문10답
  → (3) 답변을 바탕으로 기획서 작성 → (4) IA / UserScenarios /
  Personas 문서 분리 작성.
