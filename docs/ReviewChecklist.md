# ReviewChecklist.md

작업을 "완료"로 보고하기 전에 아래 항목을 점검한다. 해당 없는 항목은
이유를 남기고 건너뛴다.

## 문서

- [ ] 이번 작업으로 배경/범위가 바뀌었다면 `docs/ProjectContext.md`를
      갱신했는가?
- [ ] 구현 작업이었다면 `docs/Plan.md`를 먼저 작성했는가, 계획과 실제가
      달라진 부분을 갱신했는가?
- [ ] 사용자 결정이 필요한 부분을 임의로 진행하지 않고 질문했는가?
- [ ] 주요 의사결정을 `docs/DecisionLog.md`에 남겼는가?
- [ ] 작업 진행/오류/수정 내역을 `docs/WorkLog.md`에 남겼는가?
- [ ] 새로 조사한 기술적 내용이 있다면 `docs/Research.md`에 남겼는가?
- [ ] 사용자의 주요 프롬프트 원문을 `docs/PromptLog.md`에 저장했는가?
- [ ] IA / UserScenarios / Personas 변경이 있다면 각 독립 문서에
      반영했는가(서로 합치지 않았는가)?

## 코드 품질

- [ ] 타입체크(`npx tsc --noEmit`)가 통과하는가?
- [ ] 빌드(`npm run build`)가 성공하는가?
- [ ] 불필요한 임시 파일(스모크 테스트 스크립트, 스크린샷 등)을
      정리했는가?
- [ ] `.gitignore`에 민감 정보/생성 파일(DB, 업로드 등)이 반영되어
      있는가?

## 동작 검증

- [ ] 정적 검증뿐 아니라 실제로 앱을 구동해 golden path를 확인했는가
      (가능하면 브라우저 자동화 등으로 직접 클릭/입력까지 확인)?
- [ ] 계산/로직이 있는 경우 결과를 수기 계산 등으로 교차 검증했는가?
- [ ] 콘솔/서버 로그에 에러가 없는가?

## 커밋/공유 전 확인 (해당 시)

- [ ] `git status`로 의도치 않은 파일이 포함되지 않았는지 확인했는가?
- [ ] 비밀정보(.env 등)가 커밋 대상에 포함되지 않았는가?

- [x] 정산 결과 화면에 **적용 환율 + 기준 표시** 추가(예: "1 JPY =
