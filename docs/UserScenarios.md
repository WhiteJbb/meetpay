# 사용자 시나리오: 모임 더치페이 정산 웹

기획서([PlanningDoc.md](./PlanningDoc.md))와 페르소나([Personas.md](./Personas.md))를
바탕으로 정리한 주요 사용 흐름.

## 시나리오 1: 모임 생성 및 지출 기록 (주최자 - 김지현)

1. 지현은 친구 5명과 저녁 모임을 가진 후, 서비스에 접속해 로그인 없이
   "새 모임 만들기"를 클릭한다.
2. 모임명("금요일 저녁 모임")을 입력하고, 참여자 5명의 이름과 이메일을
   등록한다.
3. 저녁 식사비(지현이 결제, 전원 참여)를 지출 항목으로 등록한다.
4. 이어서 노래방 비용(참여자 3명만 참여, 다른 친구가 결제)을 등록하며
   참여자 체크박스에서 2명을 제외한다.
5. 모든 지출 등록 후 "정산하기"를 눌러 최종 이체 목록(누가 누구에게
   얼마)을 확인한다.
6. 공유 링크를 생성해 단체 대화방에 전달한다.

## 시나리오 2: 다국통화 정산 (해외여행 참여자 - 박민수)

1. 민수와 친구 3명은 여행 중 발생한 지출을 그때그때 기록하기로 한다.
2. 첫날 숙소비는 현지 통화(예: JPY)로, 둘째 날 저녁은 원화(KRW) 카드로
   결제해 각각 다른 통화로 지출 항목을 등록한다.
3. 영수증 사진이 있는 항목은 선택적으로 첨부해 나중에 확인할 수 있게
   한다.
4. 여행이 끝난 후 "정산하기"를 누르면 모든 지출이 기준 통화로 환산되어
   최종 이체 목록이 계산된다.
5. 민수는 결과를 공유 링크로 받아 확인하고, 실제 송금은 계좌 정보를
   보고 직접 이체한다(서비스는 송금 자체를 대행하지 않음).

## 시나리오 3: 결과만 확인하는 참여자 (이수아)

1. 수아는 총무가 보낸 공유 링크를 대화방에서 클릭한다.
2. 로그인 없이 바로 정산 결과 화면이 열리고, 모바일 화면에서도 표
   레이아웃이 깨지지 않고 잘 보인다.
3. 수아는 자신이 얼마를 누구에게 보내야 하는지 확인하고, 계좌번호를
   복사해 은행 앱으로 이동해 송금한다.

## 시나리오 4: 모임 데이터 만료

1. 정산이 끝난 모임은 특별한 조치 없이 방치되면, 정해진 보관 기간이
   지난 후 자동으로 삭제된다.
2. 결과 화면에는 "이 모임 데이터는 YYYY-MM-DD 이후 자동 삭제됩니다"와
   같은 안내가 표시되어, 필요한 경우 사용자가 미리 캡처하거나 별도
   기록을 남길 수 있게 한다.
