# 사용자 시나리오: 모임 더치페이 정산 웹

기획서([PlanningDoc.md](./PlanningDoc.md))와 페르소나([Personas.md](./Personas.md))를
바탕으로 정리한 주요 사용 흐름.

## 시나리오 1: 모임 생성 및 지출 기록 (주최자 - 김지현)

1. 지현은 친구 5명과 저녁 모임을 가진 후, 서비스에 접속해 로그인 없이
   "새 모임 만들기"를 클릭한다.
2. 모임명("금요일 저녁 모임")을 입력하고, 참여자 5명의 이름을
   등록한다(이메일·계좌는 선택 입력, 동명이인이 있을 때만 이메일이
   필수로 전환된다).
3. 저녁 식사비(지현이 결제, 전원 참여)를 지출 항목으로 등록한다.
4. 이어서 노래방 비용(참여자 3명만 참여, 다른 친구가 결제)을 등록하며
   참여자 체크박스에서 2명을 제외한다.
5. 모든 지출 등록 후 "정산 결과 보기"를 눌러 최종 이체 목록(누가
   누구에게 얼마)을 확인한다.
6. 결과 링크(열람 전용)를 복사해 단체 대화방에 전달한다.

## 시나리오 2: 다국통화 정산 (해외여행 참여자 - 박민수)

1. 민수와 친구 3명은 여행 중 발생한 지출을 그때그때 기록하기로 한다.
2. 첫날 숙소비는 현지 통화(예: JPY)로, 둘째 날 저녁은 원화(KRW) 카드로
   결제해 각각 다른 통화로 지출 항목을 등록한다.
3. 영수증 사진이 있는 항목은 선택적으로 첨부해 나중에 확인할 수 있게
   한다.
4. 여행이 끝난 후 "정산 결과 보기"를 누르면 모든 지출이 기준 통화로
   환산되어 최종 이체 목록이 계산된다(적용 환율도 함께 표시된다).
5. 민수는 결과를 공유 링크로 받아 확인하고, 실제 송금은 계좌 정보를
   보고 직접 이체한다(서비스는 송금 자체를 대행하지 않음).

## 시나리오 3: 결과만 확인하는 참여자 (이수아)

1. 수아는 총무가 보낸 공유 링크를 대화방에서 클릭한다.
2. 로그인 없이 바로 정산 결과 화면이 열리고, 모바일 화면에서도 표
   레이아웃이 깨지지 않고 잘 보인다.
3. 수아는 자신이 얼마를 누구에게 보내야 하는지 확인하고, 계좌번호를
   복사해 은행 앱으로 이동해 송금한다.

## 시나리오 4: 모임 데이터 만료

1. 정산이 끝난 모임은 특별한 조치 없이 방치되면, 정해진 보관 기간이
   지난 후 자동으로 삭제된다.
2. 결과 화면에는 "이 모임 데이터는 YYYY-MM-DD 이후 자동 삭제됩니다"와
   같은 안내가 표시되어, 필요한 경우 사용자가 미리 캡처하거나 별도
   기록을 남길 수 있게 한다.

## 메인 시나리오 (상세): 총무의 모임 생성 → 정산 마감

2026-07-07 개정판 [IA.md](./IA.md)를 기준으로, 총무(김지현 페르소나)가
모임 생성부터 정산 마감까지 가는 흐름을 화면 · 액션 · 시스템 반응
단위로 서술한다.

> **전제**: 이 서비스에는 별도의 "마감/확정" 액션이 없다. IA에서
> "정산 결과 보기"는 언제든 눌러도 되는 **조회 액션**으로 확정되었으며
> (IA §4, 이후에도 지출 추가/수정 가능), 따라서 이 시나리오의 "정산
> 마감" 국면은 **최종 결과 확인 → 결과 링크(열람 전용) 복사 → 대화방
> 공유 → 만료 안내 확인**으로 구성한다.

**상황**: 지현(총무)은 친구 5명(민재·수아·현우·보라·태균)과 금요일
저녁 모임을 가졌다. 지출은 3건 — 저녁 식사(지현 결제, 전원), 2차
카페(민재 결제, 전원), 노래방(수아 결제, 4명만 참여 — 보라·태균 불참).

### 국면 1: 모임 생성

| 단계 | 화면 | 총무의 액션 | 시스템 반응 |
|------|------|-------------|-------------|
| 1 | 랜딩/홈 | 로그인 없이 서비스에 접속하고 "새 모임 만들기" 클릭 | 모임 생성 화면으로 이동 |
| 2 | 모임 생성 | 모임명 "금요일 저녁 모임" 입력. 참여자 6명(지현·민재·수아·현우·보라·태균)의 이름을 등록. 동명이인이 없어 이메일은 입력하지 않고, 받을 돈이 있을 자신의 계좌 정보만 선택 입력 | 이름만으로 참여자 등록 허용(이메일 필수 전환 없음). "모든 지출은 KRW 기준으로 환산되어 정산됩니다" 기준 통화 안내 표시 |
| 3 | 모임 생성 | "모임 만들기"를 눌러 생성 완료 | 모임 홈(대시보드)으로 이동. 참여자 6명 목록, 지출 총액 0원(KRW), 만료 예정일 안내 배너 표시 |
| 4 | 모임 홈(대시보드) | 링크 분실에 대비해 "모임 링크 복사 (편집 가능)" 클릭 후 개인 메모에 보관 | 편집 가능 링크가 클립보드에 복사되고 "복사됨" 피드백 표시. "이 링크를 받은 사람은 지출을 수정할 수 있어요" 안내 문구로 링크 성격을 명시 |

### 국면 2: 지출 기록 (중간 점검 포함)

| 단계 | 화면 | 총무의 액션 | 시스템 반응 |
|------|------|-------------|-------------|
| 5 | 모임 홈(대시보드) | "지출 추가" 클릭 | 지출 추가/수정/삭제 화면(새 입력 폼)으로 이동. 지출일은 기본값 오늘, 참여자 분담 체크박스는 전원 선택 상태 |
| 6 | 지출 추가/수정/삭제 | 저녁 식사비 입력: 지출자 지현, 금액 180,000 + 통화 KRW, 항목명 "저녁 식사", 지출일 기본값(오늘) 유지, 분담 전원(6명) 유지, 영수증 사진 첨부 | 통화가 KRW이므로 환산 미리보기는 표시되지 않음(비KRW 통화를 선택하면 입력 즉시 "≈ N KRW" 표시). 영수증 업로드 완료 표시 |
| 7 | 지출 추가/수정/삭제 | "저장 후 계속 추가" 클릭 | 지출을 저장하고, 모임 홈으로 돌아가지 않은 채 새 입력 폼을 표시 |
| 8 | 지출 추가/수정/삭제 | 카페비 입력: 지출자 민재, 42,000 KRW, 항목명 "2차 카페", 전원 분담 → "저장" 클릭 | 저장 후 모임 홈(대시보드)으로 복귀. 지출 목록에 2건 표시("저녁 식사" 항목에 영수증 첨부 아이콘), 지출 총액 222,000 KRW과 참여자별 순 잔액 요약 갱신 |
| 9 | 모임 홈(대시보드) | 중간 점검을 위해 "정산 결과 보기" 클릭 | 정산 결과 화면(편집 모드)으로 이동. 현재까지의 참여자별 순 잔액과 최소 이체 목록 표시. 조회 액션일 뿐이므로 아무것도 마감·잠금되지 않음 |
| 10 | 정산 결과 | 이체 목록을 훑다가 노래방비가 빠진 것을 발견, "모임 홈으로 돌아가기" 클릭(모임 링크 진입이라 노출됨) | 모임 홈(대시보드)으로 복귀. 기록된 지출과 잔액은 그대로 유지 |
| 11 | 모임 홈(대시보드) | "지출 추가" 클릭 | 지출 추가/수정/삭제 화면(새 입력 폼)으로 이동 |
| 12 | 지출 추가/수정/삭제 | 노래방비 입력: 지출자 수아, 60,000 KRW, 항목명 "노래방", 분담 체크박스에서 노래방에 가지 않은 보라·태균 체크 해제(4명 부담) → "저장" 클릭 | 부분 참여 지출로 저장 후 모임 홈으로 복귀. 지출 3건, 총액 282,000 KRW, 순 잔액 요약이 부분 분담을 반영해 갱신. 지출 목록에 "노래방 — 참여 4명" 표시 |

### 국면 3: 정산 마감 (최종 확인 · 공유)

| 단계 | 화면 | 총무의 액션 | 시스템 반응 |
|------|------|-------------|-------------|
| 13 | 모임 홈(대시보드) | 지출 총액(KRW 환산)과 참여자별 순 잔액 요약으로 마감 전 검산 후 "정산 결과 보기" 클릭 | 정산 결과 화면(편집 모드)으로 이동. 참여자별 순 잔액(낸 돈 - 부담액)과 최소 이체 목록("A가 B에게 OOO원 이체") 표시. 받는 사람이 지현인 줄에는 등록된 계좌 + 복사 버튼 표시, 계좌 미등록 참여자 줄에는 "계좌 미등록" 표시. 적용 환율 안내 영역("고정 참고 환율 — 실제 결제일 환율과 다를 수 있습니다")과 데이터 만료 안내 배너도 함께 표시 |
| 14 | 정산 결과 | 참여자 이름 목록에서 "지현" 선택(내 항목 강조) | 지현이 포함된 이체 줄만 강조/필터되어, 자신이 받을 금액을 한눈에 확인 |
| 15 | 정산 결과 | "결과 링크 복사 (열람 전용)" 클릭 | 열람 전용 링크가 클립보드에 복사되고 "복사됨" 피드백 표시. "이 링크로는 결과만 볼 수 있어요" 문구로 편집 불가 링크임을 재확인 |
| 16 | (외부) 단체 대화방 | 복사한 결과 링크를 대화방에 붙여넣어 공유 | 링크를 받은 친구들은 클릭 시 정산 결과(읽기 전용 모드)로 직행. 읽기 전용 모드에서는 "모임 홈으로 돌아가기" 버튼이 노출되지 않아 지출을 수정할 수 없고, 계좌 복사로 바로 이체 가능 |
| 17 | 정산 결과 | 데이터 만료 안내 배너 확인 후 정산 마무리 | "이 모임 데이터는 YYYY-MM-DD 이후 자동 삭제됩니다" 안내 표시(만료는 마지막 활동 시점 기준으로 기산). 지현은 필요 시 결과를 캡처해 보관할 수 있음을 인지하고 마감 |

### 참고

- 본 시나리오는 전 지출이 KRW라 "≈ N KRW" 환산 미리보기와 적용 환율
  표의 환산 내역이 실제로 등장하지 않는다. 다국통화 환산·환율 표 확인
  흐름은 시나리오 2(다국통화 정산) 참고.
- 단계 9~12가 보여주듯 "정산 결과 보기"는 마감이 아닌 조회 액션이며,
  결과 확인 후에도 자유롭게 지출을 추가/수정할 수 있다(IA §4의 버튼명
  변경 취지).
