Design System · v1

모임 정산 웹 디자인 시스템

더치페이 정산의 핵심은 "숫자를 빠르고 명확하게 확인하는 것". 3장의 레퍼런스에서 공통으로 읽히는 무드 — 정돈된 화이트 카드, 절제된 포인트 컬러, 둥근 형태 — 를 기준으로 색·타이포·컴포넌트·레이아웃 규칙을 정리했다.

무드보드

00 / Moodboard

사용자가 고른 3장의 레퍼런스와 각 이미지에 남긴 코멘트. "좋다"고 표시한 요소는 살리고, "투박하다"고 표시한 요소는 다듬는 방향으로 디자인 결정을 내렸다.

무드는 채택 / 마감은 정제

민트-다크 대비 분위기와 큼직한 숫자 키패드는 좋지만, 두꺼운 아이콘·거친 도형 마감은 "투박함"으로 지적됨 → 얇은 보더, 섬세한 그림자, 정제된 아이콘으로 대체.

컴포넌트 클린함 채택

블랙×네온 컴포넌트의 정돈된 여백, 큰 잔액 숫자, 필 형태 버튼의 깔끔함을 채택. 색은 톤을 낮춘 그린으로 대체해 과감함보다 신뢰감을 우선.

카드 위 화이트 서피스 채택

은은한 배경색 위에 화이트 카드가 얹히는 레이어 구조, 아바타 칩, 리스트형 내역 UI를 기본 레이아웃 문법으로 채택.

01

정돈된 신뢰감

화려함보다 정산 결과를 믿을 수 있다는 인상. 여백과 얇은 보더로 무게감을 낮춘다.

02

절제된 포인트 컬러

단일 그린 액센트만 CTA·긍정 잔액에 사용하고 나머지는 무채색으로 절제한다.

03

친근한 라운드감

필 버튼, 둥근 카드, 원형 아바타로 딱딱한 정산 도메인을 부드럽게 만든다.

04

정렬된 숫자 가독성

금액·환율·날짜는 모노스페이스로 자릿수를 맞춰 한눈에 검산할 수 있게 한다.

05

여백 있는 카드 레이어

은은한 배경 위에 화이트 카드를 얹어 정보 영역을 명확히 분리한다.

7. 모달

07 / Modal

삭제 등 되돌리기 어려운 액션의 확인 단계에만 사용. 배경은 어둡게 딤 처리하지 않고 surface-alt로 낮은 대비만 준다.

참여자를 삭제할까요?
"박민수"는 이미 3건의 지출에 연결되어 있어요. 연결된 지출이 있으면 삭제할 수 없어요.

14. 로딩 상태

14 / Loading State

지출 목록·정산 결과 계산 중 등 데이터를 기다리는 동안 실제 레이아웃과 같은 크기의 스켈레톤을 보여준다.

15. 빈 상태

15 / Empty State

아직 데이터가 없는 상태에서 다음 행동을 바로 제안한다.

🧾

아직 등록된 지출이 없어요

첫 지출을 추가하면 참여자별 정산 결과를 바로 확인할 수 있어요.

16. 에러 상태

16 / Error State

필드 단위 에러와 페이지 단위 에러를 구분한다. negative 컬러만 사용하고 accent는 섞지 않는다.

동명이인이 있어요. 이메일을 다시 확인해 주세요.
⚠️
정산 결과를 불러오지 못했어요잠시 후 다시 시도해 주세요.

17. 접근 권한 없음

17 / Permission Denied

링크 만료(90일 보관 정책) 또는 잘못된 토큰으로 접근했을 때 보여주는 전체 화면 상태.

🔒

링크가 만료되었거나 잘못된 주소예요

모임 데이터는 마지막 활동으로부터 90일이 지나면 자동 삭제돼요. 모임 주최자에게 최신 링크를 요청해 주세요.
