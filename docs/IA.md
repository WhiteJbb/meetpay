# 정보 구조 (IA): 모임 더치페이 정산 웹

기획서([PlanningDoc.md](./PlanningDoc.md))를 바탕으로 정리한 화면 구조와
화면 간 이동 흐름.

## 1. 화면 목록

| 화면 | 설명 | 접근 방식 |
|------|------|-----------|
| 랜딩/홈 | 서비스 소개 + "새 모임 만들기" 진입점 | 누구나 접근 가능 |
| 모임 생성 | 모임명, 참여자(이름+이메일) 등록 | 누구나 접근 가능 |
| 모임 홈(대시보드) | 모임 정보, 참여자 목록, 지출 목록, "지출 추가"/"정산하기" 진입점 | 모임 공유 링크 소지자 |
| 지출 추가/수정 | 지출자, 금액, 통화, 항목명, 참여자 분담 선택, 영수증 첨부 | 모임 공유 링크 소지자 |
| 정산 결과 | 참여자별 순 잔액, 최소 이체 목록("A → B, 금액"), 보관 만료 안내 | 모임 공유 링크 소지자 (열람 전용 링크 포함) |
| 결과 공유 링크(읽기 전용) | 정산 결과만 보여주는 열람용 화면 | 공유 링크 소지자 누구나 |

## 2. 화면 간 이동 흐름

```
랜딩/홈
  └─(새 모임 만들기)→ 모임 생성
                          └─(생성 완료)→ 모임 홈(대시보드)
                                          ├─(지출 추가)→ 지출 추가/수정
                                          │                 └─(저장)→ 모임 홈(대시보드)
                                          ├─(지출 항목 클릭)→ 지출 추가/수정(수정 모드)
                                          └─(정산하기)→ 정산 결과
                                                          └─(공유 링크 생성/복사)→ 결과 공유 링크(읽기 전용)
```

- 참여자가 공유 링크로 접속하는 경로: 링크 → 모임 홈(대시보드) 또는
  정산 결과 화면으로 바로 진입(링크 종류에 따라 결정, 상세 설계
  단계에서 확정).

## 3. 주요 화면 구성 요소

### 모임 홈(대시보드)
- 모임명, 만료 예정일(안내 배너)
- 참여자 목록(이름)
- 지출 목록(항목명, 금액+통화, 지출자, 참여 인원 수)
- "지출 추가" 버튼, "정산하기" 버튼
- "공유 링크 복사" 버튼

### 지출 추가/수정
- 지출자 선택(참여자 목록 중)
- 금액 입력 + 통화 선택
- 항목명, 지출일
- 참여자 분담 체크박스 목록(기본 전체 선택)
- 영수증 이미지 첨부(선택, 업로드)

### 정산 결과
- 참여자별 순 잔액 요약(낸 돈 - 부담액)
- 최소 이체 목록: "A가 B에게 OOO원(환산 기준 통화) 이체"
- 원본 통화별 지출 내역 참고 링크(다국통화 확인용)
- 데이터 만료 안내 배너
- "공유 링크 복사" 버튼

## 4. 참고 사항

- 반응형 지원(모바일/PC 동등)을 전제로 하므로, 지출 목록/정산 결과의
  표 형태는 모바일에서 카드형으로 전환되는 등 레이아웃 적응이
  필요하다(상세 UI 설계 단계에서 확정).
- 로그인이 없는 익명 서비스이므로 "누가 이 화면에 접근할 수 있는가"는
  전적으로 링크 소지 여부로 결정된다. 이 특성은 UI 문구(예: "이 링크를
  아는 사람은 누구나 볼 수 있어요")로 사용자에게 안내되어야 한다.
