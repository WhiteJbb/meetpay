# 정보 구조 (IA): 모임 더치페이 정산 웹

기획서([PlanningDoc.md](./PlanningDoc.md))를 바탕으로 정리한 화면 구조와
화면 간 이동 흐름.

> **개정 이력**: 2026-07-07 페르소나 서브에이전트 검토(지현/민수/수아)
> 결과를 사용자 승인 하에 전체 반영. 검토 원문은
> [ReviewChecklist.md](./ReviewChecklist.md)의 "페르소나 IA 검토 결과",
> 결정 근거는 [DecisionLog.md](./DecisionLog.md) §7 참고.

## 1. 공유 링크 체계 (권한 모델)

로그인이 없는 익명 서비스이므로 **링크가 곧 권한**이다. 링크는 아래
두 종류로 확정한다.

| 링크 종류 | 권한 | 진입 화면 | 복사 위치 |
|-----------|------|-----------|-----------|
| **모임 링크** | 편집 가능 — 지출/참여자 등록·수정·삭제 | 모임 홈(대시보드) | 모임 홈의 "모임 링크 복사 (편집 가능)" 버튼 |
| **결과 링크** | 열람 전용 — 읽기만 가능 | 정산 결과(읽기 전용 모드)로 **직행** | 정산 결과의 "결과 링크 복사 (열람 전용)" 버튼 |

- 별도의 "결과 공유 링크(읽기 전용)" 화면은 **두지 않는다**. 정산 결과
  화면 하나가 진입 링크 종류에 따라 편집 모드 / 읽기 전용 모드로
  동작하며, 노출되는 버튼만 달라진다.
- 링크 복사 버튼에는 어떤 링크인지 문구로 명시한다(예: "이 링크를 받은
  사람은 지출을 수정할 수 있어요" / "이 링크로는 결과만 볼 수 있어요").
  주최자가 실수로 편집 가능 링크를 대화방에 공유하는 사고를 막기 위한
  장치다.

## 2. 화면 목록

| 화면 | 설명 | 접근 방식 |
|------|------|-----------|
| 랜딩/홈 | 서비스 소개 + "새 모임 만들기" 진입점 | 누구나 접근 가능 |
| 모임 생성 | 모임명, 참여자 등록(이름 필수 / 이메일·계좌는 선택), 기준 통화 안내 | 누구나 접근 가능 |
| 모임 홈(대시보드) | 모임 정보, 참여자 목록+관리, 지출 목록, 총액·순 잔액 요약, "지출 추가"/"정산 결과 보기" 진입점 | 모임 링크 소지자 |
| 지출 추가/수정/삭제 | 지출자, 금액+통화(환산 미리보기), 항목명, 지출일(기본값 오늘), 참여자 분담 선택, 영수증 첨부 | 모임 링크 소지자 |
| 정산 결과 | 참여자별 순 잔액, 최소 이체 목록(계좌 표시/복사 포함), 적용 환율 안내, 내 항목 강조, 보관 만료 안내 | 모임 링크 소지자(편집 모드) / 결과 링크 소지자(읽기 전용 모드) |

- 참여자 관리(추가/수정/삭제)는 별도 화면이 아니라 **모임 홈 내
  섹션**으로 둔다(소규모 모임 규모에 맞춰 화면 수 최소화).

## 3. 화면 간 이동 흐름

```
랜딩/홈
  └─(새 모임 만들기)→ 모임 생성
                          └─(생성 완료)→ 모임 홈(대시보드)
                                          ├─(지출 추가)→ 지출 추가/수정/삭제
                                          │                 ├─(저장)→ 모임 홈(대시보드)
                                          │                 └─(저장 후 계속 추가)→ 지출 추가/수정/삭제(새 입력 폼)
                                          ├─(지출 항목 클릭)→ 지출 추가/수정/삭제(수정 모드, 삭제 버튼 포함)
                                          ├─(참여자 관리)→ 모임 홈 내 참여자 섹션(추가/수정/삭제)
                                          └─(정산 결과 보기)→ 정산 결과
                                                                ├─(모임 홈으로 돌아가기)→ 모임 홈(대시보드) ※ 모임 링크 진입 시에만 노출
                                                                └─(결과 링크 복사)→ 대화방 등 외부 공유

[링크로 진입하는 경로 — 확정]
  모임 링크(편집 가능)  → 모임 홈(대시보드)
  결과 링크(열람 전용)  → 정산 결과(읽기 전용 모드) 직행
```

- 여행처럼 지출을 몰아서 입력하는 경우를 위해 "저장 후 계속 추가"
  분기를 둔다(매번 대시보드로 복귀하는 왕복 제거).
- 정산 결과에서 오류를 발견하면(예: 분담 인원 누락) "모임 홈으로
  돌아가기"로 즉시 수정 흐름에 진입할 수 있다.

## 4. 주요 화면 구성 요소

### 모임 생성
- 모임명 입력
- 참여자 등록 목록
  - **이름(필수)** — 실명 사용
  - **이메일(선택)** — 같은 이름의 참여자가 2명 이상일 때만 구분을
    위해 필수로 전환(동명이인이 없으면 요구하지 않음)
  - **계좌 정보(선택)** — 은행/계좌번호. 정산 결과에서 "받을 사람"
    계좌 표시용. 나중에 참여자 관리에서도 입력/수정 가능
- 기준 통화 안내(현재 KRW 고정): "모든 지출은 KRW 기준으로 환산되어
  정산됩니다" 문구 표시

### 모임 홈(대시보드)
- 모임명, 만료 예정일(안내 배너), 기준 통화 표시(KRW)
- 참여자 섹션: 목록(이름) + **참여자 추가/수정/삭제** 액션
  - 이미 지출에 포함된 참여자 삭제 시 처리 정책은 상세 설계 단계에서
    결정(PlanningDoc 4.1)
- 지출 목록: 항목명, 금액+통화, 지출자, 참여 인원 수,
  **영수증 첨부 여부 아이콘**(클릭 시 영수증 열람)
- **지출 총액(기준 통화 환산 합계)** — 주최자의 마감 전 검산용
- **참여자별 순 잔액 간단 요약** — 중간 확인을 위해 매번 정산 결과로
  이동할 필요 없게 함
- "지출 추가" 버튼, **"정산 결과 보기"** 버튼
  - 버튼명을 "정산하기"에서 변경: 언제든 눌러도 되는 **조회 액션**이며
    되돌릴 수 없는 마감이 아니고, 이후에도 지출 추가/수정이 가능함.
    데이터 만료 기산과도 무관함(만료는 마지막 활동 시점 기준)
- "모임 링크 복사 (편집 가능)" 버튼

### 지출 추가/수정/삭제
- 지출자 선택(참여자 목록 중)
- 금액 입력 + 통화 선택
  - **기준 통화 환산 미리보기**: 입력 즉시 "≈ 108,000 KRW"처럼 표시.
    통화 선택 실수(JPY를 KRW로 저장 등)를 입력 시점에 발견하게 함
- 항목명, 지출일(**기본값 오늘**, 변경하지 않아도 저장 가능)
- 참여자 분담 체크박스 목록(기본 전체 선택)
- 영수증 이미지 첨부(선택, 업로드) — 첨부된 영수증은 이 화면과 지출
  목록 아이콘에서 열람 가능
- 저장 버튼 + **"저장 후 계속 추가"** 버튼
- 수정 모드에서는 **삭제 버튼**(확인 단계 포함) 노출 — 중복 등록 등
  실수를 0원 수정 같은 편법 없이 정리 가능하게 함

### 정산 결과
- 참여자별 순 잔액 요약(낸 돈 - 부담액)
- 최소 이체 목록: "A가 B에게 OOO원(기준 통화) 이체"
  - 각 줄에 **받는 사람(B)의 계좌 표시 + 복사 버튼**(계좌가 등록된
    경우). 미등록이면 "계좌 미등록" 표시
- **내 항목 강조**: 참여자 이름 선택 시 해당 참여자가 포함된 이체
  줄만 강조/필터. 로그인이 없어 시스템이 "나"를 모르므로, 열람자가
  자기 금액을 한눈에 찾기 위한 장치
- **적용 환율 안내**: 사용된 환율 표(예: "1 JPY = 9 KRW")와 기준
  명시("고정 참고 환율 — 실제 결제일 환율과 다를 수 있습니다").
  환산 근거를 보여 재검산 없이 금액을 신뢰할 수 있게 함
- 원본 통화별 지출 내역 참고(다국통화 확인용) — **영수증 열람 포함**,
  읽기 전용 모드 진입자도 열람 가능
- 데이터 만료 안내 배너
- "결과 링크 복사 (열람 전용)" 버튼
- "모임 홈으로 돌아가기" 버튼 — **모임 링크(편집 모드) 진입 시에만
  노출**, 읽기 전용 모드에서는 미노출

## 5. 참고 사항

- 반응형 지원(모바일/PC 동등)을 전제로 하므로, 지출 목록/정산 결과의
  표 형태는 모바일에서 카드형으로 전환되는 등 레이아웃 적응이
  필요하다(상세 UI 설계 단계에서 확정).
- 로그인이 없는 익명 서비스이므로 "누가 이 화면에 접근할 수 있는가"는
  전적으로 링크 소지 여부로 결정된다(§1 링크 체계 참고). 이 특성은 UI
  문구(예: "이 링크를 아는 사람은 누구나 볼 수 있어요")로 사용자에게
  안내되어야 한다.
- 계좌 정보는 선택 입력 개인정보 필드다. 공유 링크(추측 불가능한
  UUID) 뒤에서만 노출되며, 모임 데이터 만료 시 함께 삭제된다
  (PlanningDoc §5 보안 항목과 동일 원칙 적용).
